最近发现自己的记忆力越来越差了，上礼拜五想了几个比较有用的工具打算这礼拜在组里分享一下，结果有一个 UI 工具今晚回忆的时候，死活想不起来了。年纪大了，还是得经常做笔记才行，想到什么第一时间记下来，不能再依赖年轻时候那种长效记忆能力了。我把自己能记住的三个列了一下：

1. [sourcebot][0]，这是一款代码搜索工具。由于我目前所在的平台部门需要供应通用组件，我经常需要在众多 git 仓库中搜索组件的使用情况。Gitlab 自带的搜索功能在某些情况下并不够用——例如它无法搜索小于号等特定字符。Sourcebot 弥补了 Gitlab 的这一不足，帮助我们更快速地定位代码。
2. [jest-styled-components][1]，这是一款增强 Jest 功能的优秀工具。其中的`modifier`功能尤其出色，帮助临时删除 hover 属性——这是我日常工作中最常使用的功能之一。
3. [`git worktree`][2]，为 git repo 创建一份完整的拷贝，这样在切换分支时无需重新安装和构建整个项目。很适合具有 long-lived feature 开发的状况。

# 2025.01.21

今日 leetcode：2715

昨天忘记的一个工具，今天终于想起来了——是 zx，顺便把之前用到过的 turborepo 和 commitlint 也写一下，周五跟小朋友们一讲内容就很充实了。

* [zx][3]：Google 出品，必属精品。它为 JavaScript 提供了调用 bash 命令的途径，特别适合在 CI 环境中使用。我曾经遇到一个场景，需要从多个 package 中复制 markdown 文件到特定目录，单写 bash 或是 js 都有点麻烦，用 zx 正好解决了我的痛点。
* [turborepo][4]: 专为 mono repo 项目设计的构建工具。它通过合理编排子项目的构建顺序，实现工作流程形式，从而最大限度地利用 CPU 资源，提高构建速度。当然，在实际运行中，它确实会占用大量 CPU 资源，一旦启动 turbo build，整个系统都会卡顿。
* [commitlint][5]: 这是一款对 git message 命名进行限制的工具。类似于 ESLint 之于 JavaScript 和 Stylelint 之于 CSS 的地位。虽然 commitlint 相对小众，但在前端工程化中也是不可或缺的重要工具。
